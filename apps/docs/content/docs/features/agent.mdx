# The Hermes Core Layer

The heart of AGIAAS is the `@agiaas/agent`. This layer is a specialized implementation of the [Hermes Agent](https://hermes-agent.nousresearch.com/docs/), an autonomous system designed by **Nous Research**.

## Powered by Hermes Agent

AGIAAS leverages the [Hermes Agent's core philosophy](https://hermes-agent.nousresearch.com/docs/): it's not a tethered copilot, but a fully autonomous entity that gets more capable the longer it runs.

## The 6-Step Pipeline

When triggered by a GitHub event, the agent follows this autonomous pipeline:

### ① Monitor
The `@agiaas/webhook-server` receives events from GitHub, Sentry, Vercel, or Cloudflare. System stats (CPU, memory) are also checked for anomalies.

### ② Reason — Mixture-of-Agents Debate
Multiple AI perspectives debate the root cause using `mixture_of_agents_debate()`. This multi-agent reasoning approach produces deeper analysis than single-model inference.

### ③ Research
The agent combines two research strategies:
- **Web Research**: Searches for similar issues, known bugs, and solutions online
- **Session Research**: Reviews historical incident data from previous sessions

### ④ Remediate — Parallel Execution
Fix tasks are executed in parallel via `parallel_remediation()`:
- Code patching via the Hermes `EditFile` skill
- Service restarts via the `RunCommand` skill
- Cache clearing and configuration fixes

### ⑤ Report — Telegram UI Cards
A structured notification is sent via Telegram with:
- Incident summary and root-cause analysis
- Generated code diff
- Real-time cost tracking (tokens + USD)
- One-tap action buttons: ✅ Approve / ❌ Reject / 💬 Re-Analyze

### ⑥ Learn — Runbook Generation
Every resolved incident automatically generates a runbook via `generate_runbook()` containing:
- Incident title and timestamp
- Root cause analysis
- Symptoms observed
- Resolution steps
- Lessons learned

## Dynamic Model Selection

AGIAAS features a **3-tier model priority system** that lets you configure AI models per repository:

```
Priority: Project Config  →  Global Default  →  API Key Auto-Detect
              ↓                   ↓                    ↓
     Dashboard per-repo     Global config       Nous key → Hermes-4-405B
     model setting          from DB             OpenRouter → Claude-3.5-Sonnet
```

### Supported Models

| Model | Provider | Best For |
| :--- | :--- | :--- |
| **Hermes-4-405B** | Nous Research (direct) | Default for Nous API keys. Tool-calling, autonomous reasoning |
| **Hermes-4-70B** | Nous Research (direct) | Lighter Nous model for faster triage |
| **Claude-3.5-Sonnet** | OpenRouter | Default for OpenRouter keys. Deep reasoning |
| **Gemini Flash** | OpenRouter | Cost-effective initial triage |

### Model Routing Logic

The agent automatically selects the right provider based on your API key:
- **`NOUS_API_KEY`** (starts with `sk-2yd`) → routes to `inference-api.nousresearch.com`
- **`OPENROUTER_API_KEY`** → routes to `openrouter.ai/api/v1`

You can override the model for each repository from the Dashboard, and the system handles API prefix normalization automatically.

> [!TIP]
> Use Hermes-4-70B or Gemini Flash for initial triage and reserve Claude-3.5-Sonnet for complex root-cause analysis to optimize costs.

## Custom Tools

AGIAAS extends the Hermes Agent with custom tools:

| Tool | Description |
| :--- | :--- |
| `send_message` | Sends structured UI Cards to Telegram with branding, cost tracking, and level-based styling |

The `send_message` tool supports multiple levels: `info`, `success`, `error`, and `incident` — each with distinct visual styling.
