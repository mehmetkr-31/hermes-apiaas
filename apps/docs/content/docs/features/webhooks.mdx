# Webhook Integrations

Hermes can process events from multiple providers to diagnose and fix issues automatically.

## 1. GitHub Integration

The primary source of events. Hermes tracks issues, PRs, and workflow failures.

- **Payload URL**: `[YOUR_TUNNEL_URL]/github/webhook`
- **Content Type**: `application/json`
- **Events**:
    - `Issues` (opened)
    - `Pull requests` (opened, synchronize)
    - `Workflow runs` (completed - detects failures)
    - `Pushes` (tracks changes for context)

> [!TIP]
> If you are using the Webhook Server, it will attempt to synchronize these settings automatically for your active projects.

---

## 2. Sentry Integration

Get automatic root-cause analysis for production errors.

- **Webhook URL**: `[YOUR_TUNNEL_URL]/sentry/webhook/[owner]/[repo]`
- **Setup**:
    1. Go to Sentry -> **Settings** -> **Integrations**.
    2. Search for **Webhooks** and add your unique URL.
    3. Create an **Alert Rule** in your project that triggers the webhook when a "New Issue" is created.

---

## 3. Vercel Integration

Detect deployment failures and large-scale regressions.

- **Webhook URL**: `[YOUR_TUNNEL_URL]/vercel/webhook/[owner]/[repo]`
- **Setup**:
    1. Go to Vercel Dashboard -> **Integrations**.
    2. Add a Custom Webhook.
    3. Select `Deployment Status` as the trigger event.

---

## 4. Cloudflare Integration

Monitor Pages deployments and Worker health.

- **Webhook URL**: `[YOUR_TUNNEL_URL]/cloudflare/webhook/[owner]/[repo]`
- **Setup**:
    1. Go to Cloudflare Dashboard -> **Pages**.
    2. Under **Settings** -> **Deployments**, add a Webhook.
    3. Hermes will analyze deployment failures specifically.

---

## Technical Details

All incoming webhooks are:
1. **Validated**: Signatures are checked (if available) against the secret in the database.
2. **Logged**: You can view raw payloads in the `packages/agent/hermes_data/on_call_logs` directory.
3. **Dispatched**: The intelligent agent is triggered in the background to avoid timeout issues with the providers.
